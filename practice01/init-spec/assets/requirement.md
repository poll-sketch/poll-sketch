# 提示词工程讲义需求说明书

## 1. 文档说明

| 项目项 | 内容 |
|--------|------|
| 关联规格 | spec.md v1.0 |
| 目标 | 明确讲义的内容质量、结构组织和交付标准 |
| 优先级说明 | P0（必须）、P1（强烈建议）、P2（可选） |

---

## 2. 功能需求

### 2.1 内容类需求

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| FR-01 | 讲义必须包含完整的7个核心章节（第0章至第6章+附录） | P0 |
| FR-02 | 每章必须明确标注“本章目标”（学完能做什么） | P0 |
| FR-03 | 每章末必须包含3–5道练习题，题型可含填空/修改提示词/简答 | P0 |
| FR-04 | 书末必须提供所有练习题的参考答案 | P0 |
| FR-05 | 第1–5章每章至少提供3组“✅好 vs ❌差”正反面对比示例 | P0 |
| FR-06 | 第3章必须提供至少6种任务类型的模板（写作/分析/代码/翻译/角色扮演/创意） | P0 |
| FR-07 | 第3章的每个模板必须附带“即填即用版”（读者可直接填空使用） | P0 |
| FR-08 | 第4章必须列出至少6个常见陷阱，每个陷阱配一个错误示例和修正方案 | P0 |
| FR-09 | 附录必须提供一页式速查卡，包含：通用公式 + 4原则 + 6模板 + 6陷阱 | P0 |
| FR-10 | 第6章必须包含至少3个完整的实战项目，每个项目分四步呈现（需求→设计→迭代→成品） | P0 |
| FR-11 | 术语首次出现时，必须同时给出中文和英文（如“思维链 Chain-of-Thought”） | P1 |
| FR-12 | 每章结尾必须提供“自查清单”（3–5个问题，读者自测掌握程度） | P1 |
| FR-13 | 第0章必须提供两种阅读路径图（课堂教学路径 / 完全自学路径） | P1 |
| FR-14 | 第5章进阶技巧（思维链、Few-shot等）每个技巧至少配2个示例 | P1 |
| FR-15 | 实战项目中的提示词示例应包含迭代过程（展示从初版到优化版的演变） | P1 |

### 2.2 结构类需求

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| FR-16 | 讲义正文篇幅控制在30–50页之间（不含封面、版权页、参考答案页） | P0 |
| FR-17 | 章节编号必须连续，小节使用层级编号（如 1.1, 1.2, 1.2.1） | P0 |
| FR-18 | 每章开头必须包含：本章目标（列表形式）+ 本章页数 | P0 |
| FR-19 | 每章内部应按逻辑顺序组织：概念引入 → 核心知识 → 示例 → 练习 | P0 |
| FR-20 | 正反面对比示例应采用视觉可区分的样式（如绿色✅ / 红色❌ 标识） | P1 |

### 2.3 可交付物完整性需求

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| FR-21 | 最终交付物必须包含：讲义正文 + 速查卡（可单独裁剪或截图） | P0 |
| FR-22 | 图标系统说明页（第0章内）必须统一解释所有图标：🎯要点 / ⚠️陷阱 / ✅好例 / ❌差例 / 🛠️模板 / 📝练习 / 🚀实战 | P0 |
| FR-23 | 参考答案必须与练习题编号一一对应 | P0 |

## 3. 非功能需求

### 3.1 可用性（Usability）

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| NFR-01 | 读者读完第1章后，应能写出包含指令+上下文的单轮提示词 | P0 |
| NFR-02 | 读者读完第2章后，应能识别并修正一个“模糊”或“无角色”的差提示词 | P0 |
| NFR-03 | 读者读完第3章后，应能独立使用至少4类模板完成对应任务 | P0 |
| NFR-04 | 自学读者无需外部资料即可完成第6章的3个实战项目 | P1 |

### 3.2 可读性（Readability）

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| NFR-05 | 提示词示例必须使用等宽字体（如代码块）展示 | P0 |
| NFR-06 | 长提示词（超过3行）应使用编号或分段展示，避免连成一团 | P0 |
| NFR-07 | 每页平均字数不超过500字（含示例），避免大段纯文本 | P1 |
| NFR-08 | 关键概念定义应使用加粗或边框突出显示 | P1 |
| NFR-09 | 陷阱警示内容应使用边框或底色区分于正文 | P1 |

### 3.3 可教学性（Teachability）

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| NFR-10 | 每章内容量应支持1–2课时的课堂教学（不含课堂练习时间） | P0 |
| NFR-11 | 实战项目应标注建议耗时（如“项目一：约15分钟”） | P1 |
| NFR-12 | 每章应标注“如果时间有限，可跳过的小节”（针对课堂教学） | P1 |

### 3.4 工具中立性（Neutrality）

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| NFR-13 | 所有示例必须适用于至少3种主流AI工具（如ChatGPT、Kimi、文心一言） | P0 |
| NFR-14 | 不得出现仅限特定平台的功能（如“点击生成图像按钮”） | P0 |

---

## 4. 用户体验需求

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| UX-01 | 读者应能在5秒内通过图标识别当前内容类型（示例/陷阱/模板/练习） | P0 |
| UX-02 | 速查卡应设计为可独立裁剪或截图保存的版式（一页纸） | P0 |
| UX-03 | 每章练习题的难度应递进：2道基础 + 1–2道中等 + 0–1道进阶 | P1 |
| UX-04 | 实战项目应让读者在完成后获得“可用的成品”体验（而非抽象练习） | P1 |
| UX-05 | 讲义中不应出现“显而易见”的填充内容（例如“本章很重要，请认真阅读”） | P1 |

## 6. 验收测试要点

| 测试项 | 通过标准 |
|--------|----------|
| 完整性测试 | 所有P0需求均已实现，无遗漏章节 |
| 可读性测试 | 随机抽取5页，每页平均字数≤500字（纯示例区域不计） |
| 可操作性测试 | B类受众读者按第1章操作，能在10分钟内写出有效提示词 |
| 模板有效性测试 | 第3章每个模板，填入真实任务后能在AI工具中产出合理结果 |
| 速查卡测试 | 速查卡单独一页，不依赖正文即可理解 |
