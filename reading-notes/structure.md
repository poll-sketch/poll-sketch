# 文档结构与提纲规范 (Structure Blueprint)

## 3.1 元结构原则

所有文档遵循"倒金字塔+模块化"原则：

- **顶部20%**：核心结论/主旨全景（让读者30秒内抓住重点）
- **中部60%**：分层论证/展开（每部分可独立阅读）
- **底部20%**：总结升华/行动号召（回扣开头，形成闭环）

## 3.2 通用章节骨架

```text
[封面/标题区]
  └─ 标题公式：《边城》读书心得 + 核心关键词（限20字内）

[摘要/导语区]（占全文5-10%，约150-300字）
  └─ 三要素：阅读背景（为何读）+ 核心发现（感受到什么）+ 结构预告（怎么展开）

[主体论证区]（占全文60-70%，约1800-2100字）
  └─ 采用"MECE原则"：相互独立，完全穷尽
  └─ 每章遵循"论点→论据→论证→小结"四步循环

[边界/局限区]（占全文5%，可选，约150字）
  └─ 主动声明本文未覆盖的内容（如不讨论沈从文其他作品）

[结论/行动区]（占全文10-15%，约300-450字）
  └─ 不引入新论据，仅整合前文并指向未来
```

## 3.3 题材定制模板：读书心得

### 模板B：读书心得（本文采用）

```text
1. 书籍与作者定位（约300字）
   └─ 一句话定义《边城》在现代文学史中的位置
   └─ 简要交代阅读背景（课程要求/自发阅读）

2. 核心情感梳理（约400字）
   └─ 梳理书中三段情感关系：翠翠与傩送、翠翠与爷爷、爷爷与顺顺
   └─ 指出"爱而不得"的核心悲剧结构

3. 关键意象深读（约500字）
   └─ 选取3个核心意象：渡船、白塔、虎耳草
   └─ 结合原文细读，分析象征意蕴

4. 个人经验勾连（约400字）
   └─ 作为软件工程学生，联系校园生活中"效率至上"的焦虑
   └─ 具体场景：实验室熬夜、代码审查的冷漠、算法竞争的孤独

5. 批判性思考（约300字）
   └─ 认同什么：纯朴人性的永恒价值
   └─ 质疑什么：悲剧是否不可避免？传统伦理是否也是枷锁？
   └─ 补充什么：当代语境下"边城"精神的转化可能

6. 行动转化（约200字）
   └─ 这本书如何改变了我接下来的一个具体行为
   └─ 例如：决定放慢生活节奏、尝试与长辈深入对话
```

## 3.4 段落微观结构

- **起承转合标记**：每段开头使用逻辑路标词（"具体而言"、"反观之"、"值得注意的是"）
- **信息密度**：每段不超过5个信息点，超过则拆分
- **过渡规则**：段与段之间必须有显性连接词或隐性逻辑链（禁止硬跳转）

## 3.5 长度控制公式

- **总字数**：3000字（基础值）中篇
- **复杂度系数**：单一主题(1.0)——本心得聚焦于《边城》一部作品
- **字数分配**：
  - 摘要/导语区：300字（10%）
  - 主体论证区：2100字（70%）
  - 边界/局限区：150字（5%）
  - 结论/行动区：450字（15%）
