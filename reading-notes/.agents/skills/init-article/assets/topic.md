# 主题锚定规范 (Topic Anchoring)

## 1.1 核心主题声明
- **主命题**：从《边城》看湘西传统文化中的"和谐共生"智慧，及其对当代软件工程人才服务国家数字化建设的启示
- **边界范围**：
  - 必须涵盖：《边城》中人与自然、人与人的关系模式；软件工程专业的技术伦理与社会责任；祖国数字化转型的战略需求
  - 明确排除：沈从文生平传记、《边城》的文学创作手法分析、其他文学作品对比
- **目标读者**：大二软件工程专业学生、高校思政教育工作者、关注科技伦理的青年群体

## 1.2 偏题检测机制
生成过程中，每完成一个章节需自检：
- [ ] 本节内容是否直接支撑"传统文化智慧与软件工程人才成长"这一核心关联？
- [ ] 若删除本节，主命题是否仍然完整？（如答案是"是"，则本节可能冗余）
- [ ] 本节是否引入了边界范围外的新话题？（如纯文学分析、与专业无关的个人感想）

## 1.3 题材适配示例
| 题材类型 | 主命题示例 | 边界排除示例 |
|---------|-----------|------------|
| 软件工程报告 | 本系统如何通过微服务架构解决高并发场景下的性能瓶颈 | 不讨论前端UI设计细节、不对比其他公司商业产品 |
| 读书心得（本规范） | 《边城》中"和谐共生"智慧对软件工程人才服务国家数字化建设的启示 | 不介绍作者生平年表、不讨论该书出版市场反响、不进行纯文学赏析 |
| 入党申请书 | 个人在思想/实践/学习三维度向党组织靠拢的历程与决心 | 不罗列所有获奖证书、不抱怨社会负面现象 |

## 1.4 纠偏触发词
当内容偏离时，使用以下句式强制回归：
"回到核心命题：《边城》智慧与软件工程人才成长的关联，我们需要聚焦于..."

## 1.5 专业结合锚点
本读书心得必须包含以下专业结合点：
- 翠翠与爷爷的信任关系 → 软件系统中的权限管理与信任机制
- 茶峒的和谐生态 → 分布式系统的协调与平衡
- 老船夫的坚守 → 软件工程师的职业操守与长期主义
- 湘西的传统与变迁 → 数字化转型中的文化传承与技术创新
