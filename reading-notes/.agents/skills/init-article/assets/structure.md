# 文档结构与提纲规范 (Structure Blueprint)

## 3.1 元结构原则
所有文档遵循"倒金字塔+模块化"原则：
- 顶部20%：核心结论/主旨全景（让读者30秒内抓住重点）
- 中部60%：分层论证/展开（每部分可独立阅读）
- 底部20%：总结升华/行动号召（回扣开头，形成闭环）

## 3.2 通用章节骨架
```text
[封面/标题区]
  └─ 标题公式：核心对象 + 关键动作 + 价值指向（限20字内）

[摘要/导语区]（占全文5-10%）
  └─ 三要素：背景（为何写）+ 核心发现（是什么）+ 结构预告（怎么读）

[主体论证区]（占全文60-70%）
  └─ 采用"MECE原则"：相互独立，完全穷尽
  └─ 每章遵循"论点→论据→论证→小结"四步循环

[边界/局限区]（占全文5%，可选）
  └─ 主动声明本文未覆盖的内容或存在的局限

[结论/行动区]（占全文10-15%）
  └─ 不引入新论据，仅整合前文并指向未来
```

## 3.3 题材定制模板

### 模板A：软件工程报告
```text
1. 项目背景与问题定义（痛点数据化）
2. 技术方案选型（对比维度：性能/成本/维护性）
3. 架构设计与实现（分层图+关键代码逻辑）
4. 测试验证与数据（基准测试+压力测试）
5. 部署运维与监控（CI/CD流程+告警策略）
6. 总结与演进路线（已解决/待优化/下一步）
```

### 模板B：读书心得（本规范 - 《边城》专题）
```text
1. 书籍定位与阅读背景（《边城》在当代青年精神建构中的价值，个人阅读契机）
2. 核心意象解读（茶峒世界的"和谐共生"模式：人与自然、人与人、传统与现代）
3. 专业视角重构（将文学意象映射到软件工程：信任机制、系统平衡、长期主义）
4. 国家发展需求对接（数字化转型背景下，软件工程人才应具备的人文素养与技术伦理）
5. 批判性思考（传统智慧的现代局限，技术发展的伦理困境，如何平衡创新与责任）
6. 行动转化计划（具体学习计划、实践方向、职业规划，如何践行"技术向善"）
```

### 模板C：入党申请书
```text
1. 入党动机（真实触发点，拒绝模板化表述）
2. 思想成长历程（理论学习的具体书目与心得节点）
3. 实践服务经历（时间/地点/事件/反思，STAR法则）
4. 自我剖析与不足（具体缺点+改进计划，忌"过于追求完美"）
5. 未来承诺（可量化的行动指标，而非口号）
```

## 3.4 段落微观结构
- **起承转合标记**：每段开头使用逻辑路标词（"具体而言"、"反观之"、"值得注意的是"、"从专业角度看"）
- **信息密度**：每段不超过5个信息点，超过则拆分
- **过渡规则**：段与段之间必须有显性连接词或隐性逻辑链（禁止硬跳转）
- **专业融合规则**：每章至少包含1个文学意象与1个专业概念的关联分析

## 3.5 长度控制公式
- 总字数 = 基础值 × 复杂度系数
  - 基础值：短篇(1500) / 中篇(3000) / 长篇(5000)
  - 复杂度系数：单一主题(1.0) / 对比分析(1.3) / 系统性综述(1.6)
- 本规范要求：≥3000字，复杂度系数1.3（跨学科对比分析）

## 3.6 章节字数分配建议
| 章节 | 建议字数 | 占比 |
|-----|---------|------|
| 书籍定位与阅读背景 | 400-500字 | 13-17% |
| 核心意象解读 | 600-800字 | 20-27% |
| 专业视角重构 | 800-1000字 | 27-33% |
| 国家发展需求对接 | 500-600字 | 17-20% |
| 批判性思考 | 400-500字 | 13-17% |
| 行动转化计划 | 300-400字 | 10-13% |
