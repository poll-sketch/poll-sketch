# 实验报告2：AI智能体工具调用功能实现

## 实验目的

1. **掌握工具调用机制**：学习如何为AI智能体添加工具调用能力，实现LLM与外部工具的交互。
2. **实现文件操作工具**：开发文件系统操作工具，包括列出目录、修改文件名、删除文件、新建文件和读取文件。
3. **实现网络访问工具**：开发网络访问工具，支持通过HTTP/HTTPS访问网页并获取内容。
4. **构建工具调用流程**：实现完整的工具调用流程，包括工具调用请求的生成、执行和结果处理。
5. **增强AI智能体能力**：通过工具调用扩展AI智能体的功能，使其能够执行实际的文件操作和网络访问。

## 实验过程

### 1. 环境准备

1. **创建项目结构**：在`practice02`目录中创建实验文件。
2. **配置环境变量**：确保`.env`文件已正确配置，包含LLM的API密钥和其他必要参数。

### 2. 工具函数实现

#### 2.1 文件操作工具

1. **列出目录文件** (`list_directory`):
   - 功能：列出指定目录下的所有文件和子目录
   - 参数：`directory` - 目录路径
   - 返回：包含文件信息的列表，每个文件包含名称、路径、类型等信息

2. **修改文件名** (`rename_file`):
   - 功能：修改文件或目录的名称
   - 参数：`old_path` - 原文件/目录路径，`new_name` - 新名称
   - 返回：操作结果，包括成功或错误信息

3. **删除文件** (`delete_file`):
   - 功能：删除指定文件
   - 参数：`file_path` - 文件路径
   - 返回：操作结果，包括成功或错误信息

4. **新建文件** (`create_file`):
   - 功能：新建文件并写入内容
   - 参数：`file_path` - 文件路径，`content` - 要写入的内容
   - 返回：操作结果，包括成功或错误信息

5. **读取文件** (`read_file`):
   - 功能：读取指定文件的内容
   - 参数：`file_path` - 文件路径
   - 返回：文件内容，包括成功或错误信息

#### 2.2 网络访问工具

6. **curl网络访问** (`curl_request`):
   - 功能：通过HTTP/HTTPS访问网页并返回内容
   - 参数：`url` - 访问的URL，`method` - 请求方法（默认GET），`headers` - 请求头，`data` - 请求数据
   - 返回：网页内容、状态码和响应头

### 3. 工具调用机制实现

#### 3.1 工具映射

```python
tools = {
    "list_directory": list_directory,
    "rename_file": rename_file,
    "delete_file": delete_file,
    "create_file": create_file,
    "read_file": read_file,
    "curl_request": curl_request
}
```

#### 3.2 系统提示词

定义了详细的工具调用说明，包括工具名称、参数和返回值，以及工具调用的格式示例。

#### 3.3 LLM API调用（支持工具调用）

`call_llm_with_tools`函数实现了支持工具调用的LLM API调用，包括：
- 构建符合OpenAI兼容协议的请求数据
- 发送API请求并处理响应
- 统计API调用的性能指标

#### 3.4 工具调用处理

`process_tool_calls`函数实现了工具调用的处理逻辑：
- 解析LLM生成的工具调用请求
- 调用相应的工具函数
- 处理工具执行结果
- 返回标准化的工具执行结果

### 4. 主聊天循环实现

`chat_loop`函数实现了支持工具调用的交互式聊天循环：
1. 加载环境变量
2. 初始化聊天历史，包含系统提示词
3. 获取用户输入
4. 调用LLM获取回复
5. 处理工具调用请求
6. 执行工具并获取结果
7. 将工具执行结果添加到聊天历史
8. 再次调用LLM获取最终回复
9. 管理聊天历史长度
