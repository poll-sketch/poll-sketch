# 实验报告5：AI智能体AnythingLLM文档仓库访问功能实现

## 实验目的

1. **掌握AnythingLLM集成**：学习如何集成AnythingLLM本地服务，实现文档仓库的访问和查询。
2. **实现工具调用扩展**：开发新的工具调用功能，支持8个工具的调用，包括文件操作、网络访问、聊天历史搜索和AnythingLLM查询。
3. **优化工具调用机制**：完善工具调用流程，确保工具调用的正确性和可靠性。
4. **增强AI智能体能力**：通过集成AnythingLLM，使AI智能体能够访问本地文档仓库，提高信息获取能力。
5. **测试功能完整性**：验证AnythingLLM集成和工具调用的功能完整性。

## 实验过程

### 1. 环境准备

1. **创建项目结构**：在`practice05`目录中创建实验文件。
2. **配置环境变量**：确保`.env`文件已正确配置，包含LLM的API密钥和AnythingLLM的配置参数：
   - ANYTHINGLLM_API_KEY: AnythingLLM的API密钥
   - ANYTHINGLLM_WORKSPACE_SLUG: AnythingLLM的工作区Slug ID
3. **启动AnythingLLM服务**：确保本地AnythingLLM服务已启动，可通过`http://localhost:3001`访问。

### 2. 核心功能实现

#### 2.1 工具函数

1. **文件操作工具**：
   - `list_directory`：列出目录下的文件和子目录
   - `rename_file`：修改文件或目录的名称
   - `delete_file`：删除文件
   - `create_file`：新建文件并写入内容
   - `read_file`：读取文件内容

2. **网络访问工具**：
   - `fetch_webpage`：通过HTTP/HTTPS访问网页并获取内容

3. **聊天历史工具**：
   - `search_chat_history`：搜索聊天历史记录

4. **AnythingLLM工具**：
   - `anythingllm_query`：使用AnythingLLM查询文档仓库（新增）

#### 2.2 AnythingLLM集成

1. **AnythingLLM查询函数** (`anythingllm_query`)：
   - 功能：使用AnythingLLM查询文档仓库，获取相关文档信息
   - 参数：`message` - 查询消息
   - 处理流程：
     - 加载环境变量，获取API密钥和工作区Slug
     - 构建请求URL：`http://localhost:3001/api/v1/workspace/{workspace_slug}/chat`
     - 构建请求数据，包含查询消息
     - 使用subprocess调用curl命令发送请求
     - 处理响应，返回查询结果

2. **环境变量配置**：
   - 从`.env`文件读取AnythingLLM的API密钥和工作区Slug
   - 提供默认值和错误处理

#### 2.3 工具调用机制

1. **工具映射**：
   - 包含8个工具：列出目录、修改文件名、删除文件、新建文件、读取文件、访问网页、搜索聊天历史、AnythingLLM查询
   - 工具名称与老师的仓库保持一致

2. **系统提示词**：
   - 详细描述每个工具的功能、参数和返回值
   - 明确工具调用的格式和触发条件
   - 特别说明当用户提到"文档仓库"、"文件仓库"、"仓库"等关键词时，优先调用`anythingllm_query`工具

3. **工具调用处理**：
   - 解析LLM生成的工具调用请求
   - 调用相应的工具函数
   - 处理工具执行结果
   - 返回标准化的工具执行结果

#### 2.4 聊天记录总结

- 继承practice03的`summarize_chat_history`函数
- 当聊天轮数超过5轮或上下文长度超过3k时触发总结
- 对前70%的内容进行压缩，保留后30%的内容

### 3. 主聊天循环

`chat_loop`函数实现了支持AnythingLLM查询的交互式聊天循环：
1. 加载环境变量
2. 初始化聊天历史，包含系统提示词
3. 获取用户输入
4. **检查是否需要总结聊天记录**：
   - 当聊天轮数超过5轮时
   - 当聊天上下文长度超过3000字符时
5. 调用LLM获取回复
6. 处理工具调用请求：
   - 执行工具并获取结果
   - 将工具执行结果添加到聊天历史
   - 再次调用LLM获取最终回复
7. 管理聊天历史长度

### 4. 测试功能

`test_anythingllm`函数用于测试AnythingLLM查询功能：
1. 构建测试查询消息："仓库里有什么文档？"
2. 调用`anythingllm_query`函数
3. 输出查询结果
4. 验证测试是否成功

### 5. 实验测试

1. **测试AnythingLLM查询**：
   - 启动AnythingLLM本地服务
   - 运行`python practice05/tool_client.py`
   - 输入"仓库里有什么文档？"，观察是否自动调用AnythingLLM查询
   - 检查查询结果是否正确

2. **测试工具调用**：
   - 测试文件操作工具（列出目录、读取文件等）
   - 测试网络访问工具（访问网页）
   - 测试聊天历史搜索工具（搜索关键词）
   - 验证工具调用的正常运行

3. **测试聊天记录总结**：
   - 发送多轮消息，观察是否自动触发总结
   - 验证总结后的聊天历史是否保持连贯性

4. **测试错误处理**：
   - 测试未配置AnythingLLM参数的情况
   - 测试AnythingLLM服务未启动的情况
   - 验证错误处理的正确性
