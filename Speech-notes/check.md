# 演讲稿审核规范

## 必查项目

### 1. 内容准确性检查
- [ ] 所有数据准确无误（专家评分4.42、学习提升105.2%、Agent提升134.6%）
- [ ] 项目名称正确（《人工智能提示工程》课程讲义）
- [ ] 关键术语使用正确（LLM、RAG、Skill系统、SPEC范式）
- [ ] 无虚假或夸大表述

### 2. 结构完整性检查
- [ ] 包含问题背景、解决方案、创新点、成果数据
- [ ] 逻辑顺序正确（问题-方案-成果）
- [ ] 有清晰的开场、主体和结尾
- [ ] 段落过渡自然流畅

### 3. 字数控制检查
- [ ] 总字数不超过900字
- [ ] 各部分字数分配合理
- [ ] 无冗长表述，信息密度高

### 4. 语言风格检查
- [ ] 避免"综上所述"、"基于以上分析"等套话
- [ ] 口语化表达，适合演讲
- [ ] 使用主动语态和行动动词
- [ ] 句子简洁，平均长度不超过15字

### 5. 数据支撑检查
- [ ] 每个核心观点都有数据支撑
- [ ] 数据表述清晰（如"提升134.6%"而非"大幅提升"）
- [ ] 关键数据在演讲中突出强调

### 6. 禁忌内容检查
- [ ] 无技术细节堆砌
- [ ] 无无关个人感受
- [ ] 无模糊词汇（大概、可能）
- [ ] 无过度谦虚表达

### 7. 时间适配检查
- [ ] 按300字/分钟语速，时间控制在3分钟内
- [ ] 重点内容有足够时间展开
- [ ] 预留互动或机动时间

## 评分标准
| 维度 | 合格标准 | 不合格表现 |
|-----|---------|-----------|
| 内容 | 数据准确，逻辑清晰 | 数据错误，逻辑混乱 |
| 结构 | 完整有序，过渡自然 | 结构松散，缺乏逻辑 |
| 语言 | 简洁有力，口语化 | 冗长晦涩，书面化 |
| 字数 | ≤900字 | >900字 |
| 风格 | 自信专业 | 套话连篇，缺乏活力 |

## 修改原则
- 发现问题立即修改，确保符合所有规范
- 优先保证内容准确性和结构完整性
- 在保证质量的前提下优化语言表达
