## 通知撰写技能

本技能用于撰写、修改和润色各类通知文件。

### 格式要求
1. **标题格式**：通知不能以"通知"二字开头，必须冠以部门名称前缀
2. **部门前缀**：格式为"XX部通知"，例如"采购部通知"、"宣传部通知"、"销售部通知"等
3. **默认部门**：如果用户没有告知所在部门，使用"XX部"代替

### 使用场景
- 用户要求撰写通知时
- 用户要求修改通知时
- 用户要求润色通知时
- 用户提到"通知"、"公告"、"放假"等关键词时

### 输出格式
```
【部门名称】通知
通知正文内容...
```

### 示例
- 用户未提供部门：XX部通知
- 用户提供部门"销售部"：销售部通知
- 用户提供部门"采购部"：采购部通知

### 注意事项
1. 严格遵守标题格式要求，必须以部门名称开头
2. 确保通知内容正式、规范
3. 如果用户提供了具体部门名称，必须使用该部门名称
4. 如果用户未提供部门，统一使用"XX部"作为前缀
