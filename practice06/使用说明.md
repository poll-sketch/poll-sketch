## 功能说明

### 1. list_available_skills 函数

**功能**：读取 `.agents/skills` 目录下所有技能的配置信息

**实现细节**：
- 遍历 `.agents/skills` 目录下的所有一级子目录
- 读取每个子目录中的 `SKILL.md` 文件
- 提取 YAML front matter（`---` 标记之间的内容）
- 解析 `name` 和 `description` 字段
- 返回技能列表

**返回格式**：
```json
{
  "status": "success",
  "data": [
    {
      "name": "技能名称",
      "description": "技能描述"
    }
  ]
}
```

### 2. load_skill_content 函数

**功能**：加载指定技能的详细内容

**实现细节**：
- 接收技能名称（YAML front matter 中的 name 字段）作为参数
- 遍历所有技能目录，查找匹配的技能
- 提取 SKILL.md 文件的正文内容（YAML front matter 之后的部分）
- 返回技能正文

**返回格式**：
```json
{
  "status": "success",
  "data": "技能正文内容..."
}
```

### 3. 主聊天循环

**功能**：自动化的技能管理流程

**实现细节**：
- 每次用户提交输入时，自动调用 `list_available_skills()` 读取技能列表
- 将技能列表以 JSON 格式添加到 system prompt 中
- 更新聊天历史的系统消息
- LLM 可以根据技能列表选择合适的技能来执行任务

**System Prompt 格式**：
```
你是一个AI助手，拥有以下工具调用能力：

1. list_available_skills(): 读取可用技能列表
   返回：技能列表，每个技能包含name和description字段

2. load_skill_content(skill_name): 加载指定技能的内容
   参数：skill_name - 技能名称
   返回：技能正文内容

可用技能列表：
{
  "skills": [
    {"name":"技能名称","description":"技能描述"}
  ]
}
```

## 技能文件格式

技能文件必须遵循以下格式：

```markdown
---
name: "技能名称"
description: "技能描述"
---

## 技能正文

这里是技能的详细内容...
```

**注意事项**：
- YAML front matter 必须以 `---` 开始和结束
- `name` 和 `description` 字段是必需的
- 正文内容在第二个 `---` 之后

## 使用示例

### 1. 创建新技能

在 `.agents/skills/` 目录下创建新的子目录，例如 `skill3/`，然后创建 `SKILL.md` 文件：

```markdown
---
name: "数据分析"
description: "对数据进行分析和可视化"
---

## 数据分析技能

本技能用于数据分析，包括：
- 数据清洗
- 统计分析
- 数据可视化
```

### 2. 运行主程序

```bash
python tool_client.py
```

程序会启动交互式聊天界面，每次用户输入时都会自动更新技能列表。

### 3. 运行测试

**测试技能功能**：
```bash
python test_skills.py
```

**测试 LLM 集成**：
```bash
python test_integration.py
```

## 关键改进

1. **路径修正**：将项目根目录从 `os.path.dirname(os.path.dirname(__file__))` 改为 `os.path.dirname(__file__)`，确保正确读取 practice06 目录下的技能

2. **技能查找优化**：`load_skill_content` 函数现在会遍历所有技能目录，根据 YAML front matter 中的 name 字段查找匹配的技能，而不是直接使用目录名称

3. **错误处理**：添加了完善的错误处理机制，包括文件不存在、目录不存在等情况的处理

4. **自动化流程**：在主聊天循环中实现了自动化的技能列表更新机制

## 扩展性

本系统具有良好的扩展性：

1. **添加新技能**：只需在 `.agents/skills/` 目录下创建新的子目录和 SKILL.md 文件
2. **技能热更新**：每次用户交互时自动读取最新的技能列表
3. **工具集成**：可以轻松集成到其他工具系统中
4. **LLM 集成**：支持与各种 LLM 模型集成，实现智能技能选择

## 注意事项

1. 确保 `.env` 文件正确配置了 API_KEY 等必要参数
2. 技能文件必须使用 UTF-8 编码
3. YAML front matter 中的 name 字段必须唯一
4. 技能目录名称可以与 name 字段不同，系统会根据 name 字段进行匹配
